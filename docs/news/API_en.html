OpenLabor API specification

General

The OpenLabor APIs have being developed inside CitySDK project. They are useful for sending job and training reports and requests to cities services.
At the moment, APIs are used for sending reports/requests about job opportunities, but they are planned for being used for vacational training opportunities as well.
The following functionalities are provided by the APIs:
  • Listing of service request types and definitions of the additional attributes
  • Submitting service requests
  • Quering individual or multiple service requests and their descriptions and status
The issue reporting API is based on GeoReporting version 2, which is better known as Open311 specification. This interface is compliant with CitySDK specific enhancements to Open311 API

API Key

Applications willing to submit jobs or training offers or comments will need an API key. All other methods are open (do not require API keys).

Encoding and formats

UTF-8 is required everywhere. API supports both XML and JSON.

Language support

At the moment, the only values acceptable for the "locale" variable are en_US and it_IT. See more about multilingual issues in the general information about OpenLabor project.

API methods

OpenLabor provides RESTFul API methods according to international standard Open311:

  • Check out the service (discovery): XML o JSON
  • Check out the services (services): XML o JSON
  • Check out the details of JOBS search service: XML o JSON
  • Check out the details of the reporting job service: XML o JSON
  • Check out the details of the training opportunities search service: XML o JSON
  • Check out the details of the reporting training opportinities service: XML o JSON
